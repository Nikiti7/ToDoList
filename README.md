# Task Management System (Symfony 7)

Бэкенд-часть системы управления задачами, включающая в себя бизнес-логику очередей, REST API, консольные команды и схематичный веб-интерфейс.

## Стек технологий

* **PHP** 8.2+ (тестировалось на PHP 8.5)
* **Framework:** Symfony 7.0
* **ORM:** Doctrine ORM (Паттерн Data Mapper)
* **Database:** SQLite
* **Template Engine:** Twig (для веб-интерфейса)

## Установка и запуск

### 1. Клонирование и установка зависимостей

```bash
composer install
```

### 2. Инициализация базы данных (SQLite)

Создание файла БД и запуск миграций (включая нулевую):

```bash
php bin/console doctrine:database:create
php bin/console doctrine:migrations:migrate --no-interaction
```

### 3. Запуск локального сервера

```bash
php -S 127.0.0.1:8000 -t public
```

Проект будет доступен по адресу: `http://127.0.0.1:8000`

## Проверка функционала

### 1. Консольные команды (`bin/console`)

* Создание задачи (интерактивный режим, авто-статус `new`, вывод в JSON):

 ```bash
 php bin/console app:task:create
 ```

* Просмотр списка задач (в виде таблицы + опциональный фильтр по статусу):

 ```bash
 php bin/console app:task:list
 php bin/console app:task:list new
 ```

* Смена статуса задачи (при пустых аргументах переводит все `new` в `in_progress`):

 ```bash
 php bin/console app:task:change-status 1 in_progress
 php bin/console app:task:change-status
 ```

### 2. REST API & Логирование ошибок

Для тестирования API в корне проекта подготовлен файл api.http (совместим с VS Code REST Client / PhpStorm).

* **Глобальный обработчик ошибок**: Любые ошибки API (404, 405, 500) перехватываются и всегда возвращаются в формате `JSON`, а не HTML-страницей Symfony.

* **Логирование**: Все действия с задачами и ошибки пишутся в стандартный лог-файл: `var/log/dev.log`.

### 3. Веб-интерфейс

Полноценная таблица со списком всех задач доступна в браузере: `http://127.0.0.1:8000/tasks`

## Реализация Очереди задач (Бизнес-логика)

Вся бизнес-логика вынесена в `TaskService` и работает по следующим правилам:

* **Лимит на новые задачи**: Если в статусе `new` уже находится 5 задач, любая следующая создаваемая задача автоматически получает статус `pending`.

* Продвижение из очереди: Как только любая задача уходит из статуса `new` (переводится в `in_progress`, `done` и т.д.), самая старая задача из статуса `pending` автоматически переводится в статус `new`.
